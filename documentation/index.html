OPULENT

DOCUMENTATION

Opulent is a templating engine which strives to make page markup as beautiful and expressive as it should be. It's blazing fast, offers useful reusable component definitions and encourages well organised front end code. Opulent provides the cleanliness, readability and development speed you need for your project.

INSTALLATION

GETTING STARTED

First of all, you will need to make sure you have Ruby and Ruby Gems installed on your system. We recommend using the latest Ruby version available. Opulent is available as a module for most major web frameworks written in Ruby.

To install the opulent gem run the following command in your terminal:

gem install opulent

To use opulent inside your web application you can add it to your Gemfile.

gem "opulent"

Here's how our first web page would look like with opulent. Let's name it input.op.

doctype html 
html
  head
    title Hello Opulent! 
  body 
    #content
      | This is my first web page

To run opulent from the command line, you can use:

opulent input.op output.html

You can also use Opulent inside any Ruby Application as follows:

String (code input)

When passing a String to the constructor, the engine will proceed the code generation for the code written inside the string.

engine = Opulent.new "p Hello opulent!"
engine.render # => "<p>Hello opulent!</p>"
Symbol (path input)

If we pass a Symbol to the constructor, the engine will open the file at the path designated by the Symbol, like this: "#{:path}.op".

engine = Opulent.new :input
engine.render # => "<html><head>..."

INSTALLATION

RUBY ON RAILS

First you will need to install the Ruby On Rails implementation of Opulent that can be found in the following gem:

gem install opulent-rails

Once it’s installed, all view files with the ".html.op" extension will be compiled using Opulent. You can access instance variables in Opulent Templates the same way you do in ERB templates. Helper methods are also available in Opulent templates. For example:

class MoviesController < ApplicationController

  end
end

Rails XSS Protection

Opulent supports Rails’ XSS protection scheme, which was introduced in Rails 2.3.5+ and is enabled by default in 3.0.0+. Like in ERB, all strings printed to a Opulent template are escaped by default. Also like ERB, strings marked as HTML safe are not escaped. Opulent also has its own syntax for printing a raw string to the template using the ~ (tilde) operator.

SYNTAX

BASIC NODES

The node syntax in opulent is inspired by SASS, and the same principles apply. You have no angular brackets and every parent-child relation is based on indentation. The node name given will be the same when rendered to HTML.

There are some special cases where the node name can be missing, and in that case, the implicit node is div because it is the most widely used.

div 
      
/ => <div></div>
bread 
  tomato
  ham 
        
/ <bread><tomato></tomato><ham></ham></bread>

In Opulent the following node names are valid: lowercase, UPPERCASE, camelCase, snake_case and hyphen-case.

The preferred way to name and define nodes in opulent is hyphen-case. As a convention, we're going to use hyphen-case throughout the documentation.

validname
VALIDNAME
validName
valid_name
valid-name

SYNTAX

INLINE NODES

To encourage a compact markup, you can also use the inline operator > to write a parent-child relation between nodes.

ul 
  li > a href="http://pixevil.com" > span Pixevil
      
/ <ul>
/   <li><a href="http://pixevil.com"><span>Pixevil</span></a></li>
/ </ul>

When having direct children after declaring inline child nodes, the direct children will be inside the last declared inline node.

ul 
  li > a href="http://pixevil.com" 
    i.fa.fa-globe
    span Pixevil
      
/ <ul>
/   <li><a href="http://pixevil.com"><i class="fa fa-globe"></i><span>Pixevil</span></a></li>
/ </ul>

SYNTAX

NODE WHITESPACE

You can handle whitespace (spaces before and after a node) using the ' (apostrophe) for whitespace to the left of the node and " (quote) for whitespace to the right of the node.

The same whitespace syntax applies to all nodes, even plain text nodes.

div
div'
div"
div'"
div
      
/ <div></div> <div></div><div></div>  <div></div> <div></div>

SYNTAX

SELF CLOSING NODES

Self closing nodes from HTML can be marked up using a trailing / (forward slash) the end of the node markup. Self enclosing nodes cannot have any child nodes.

node /
/ <node>
      
node 
/ <node></node>

Opulent will automatically close the most common self-closing HTML nodes: img, link, input, meta, br, hr, area, base, col, command, embed, keygen, param, source, track and wbr.

SYNTAX

SHORTHAND ATTRUBUTES

As mentioned earlier, the shorthand syntax is similar to SASS and uses . (dot) for classes, # (hash) for ids and less common, & (ampersand) for the input node's name attribute.

The node name can be missing and is implicitly set to div, being the most commonly used node.

div#id 
/ <div id="id"></div>
      
.node-class
/ <div class="node-class"></div>
      
#node-id
/ <div id="node-id"></div>
      
input&username
/ <input name="username">

You can chain as many shorthand attributes as you want. You can use as many classes as you want for your node, but the other attributes will be replaced and only the last assigned value will show in the rendered code.

a.btn.btn-primary#click-me href="http://opulent.io" Hello Opulent! 
/ <a class="btn btn-primary" id="click-me" href="http://opulent.io">Hello Opulent!</a>

You can also pass a string as the value for the shorthand attribute. You can use interpolation on the passed string, which we will see in the next section.

a."button"
/ <a class="button"></a>

SYNTAX

WRAPPED ATTRUBUTES

The wrapped attributes syntax makes use of brackets () (round brackets) , [] (square brackets) and {} (curly brackets) to surround the atrributes of the node.

For attribution you can use either the classic = (equals) symbol or the : (colon) symbol.

div(id="id") 
/ <div id="id"></div>
      
div(id:"id") 
/ <div id="id"></div>
      
div(class="node-class")
/ <div class="node-class"></div>

div[class="example" id="node"]
/ <div class="example" id="node"></div>

The attributes can be followed by a comma, and can span multiple lines if the wrapped syntax is used.

div(class="first", class="second")
/ <div class="first second"></div>
      
div(
  class="first", 
  class="second"
)
/ <div class="first second"></div>

You can also use wrapped attributes in combination with shorthand attributes. However, the order is important: shorthand attributes need to be declared first.

a.btn(id="button" href="http://opulent.io") Hello Opulent! 
/ <a class="btn" id="button" href="http://opulent.io">Hello Opulent!</a>

SYNTAX

UNWRAPPED ATTRUBUTES

The unwrapped attributes syntax allows you to write key-value pairs inline with the node, like you would do in HTML.

Just like the wrapped attributes syntax, you can use you can use either the = (equals) symbol or the : (colon) symbol for key-value attributions.

div id="id"
/ <div id="id"></div>
      
div class="node-class"
/ <div class="node-class"></div>

div class="example" id="node"
/ <div class="example" id="node"></div>
      
div class:"example" id:"node"
/ <div class="example" id="node"></div>

Below you can see how you can combine all the three attribute syntaxes and use them seamlessly in your markup.

a.btn#button(data-attr="value") href="http://opulent.io" Hello Opulent! 
/ <a class="btn" id="button" data-attr="value" href="http://opulent.io">Hello Opulent!</a>

SYNTAX

ESCAPED ATTRUBUTES

All attributes are rendered as encoded HTML by default. If you consider the variable to be HTML Safe, then you can unescape it by using the ~ (tilde) character right after the attribution symbol.

div attr="<escaped html>"
/ <div attr="&lt;escaped html&gt;"></div>
      
div attr=~"<unescaped html>"
/ <div attr="<unescaped html"></div>

SYNTAX

INLINE TEXT

Text in opulent can be written directly inline with the node, after all the attributes have been declared.

li Hello World
/ <li>Hello World</li>

Opulent will assign child nodes to the parent as you would normally expect, with the inline text being the first direct child.

li Hello 
  span.child World
/ <li>Hello<span class="child">World</span></li>

SYNTAX

EXPLICIT TEXT

A text node in opulent is marked using a | (pipe) symbol, and can span multiple lines as long as the text is more indented than the pipe character used.

| Hello World
/ Hello World
      
| Hello Opulent!
  I'm on multiple lines.
/ Hello Opulent
/ I'm on multiple lines.

Just like the nodes, the text markup handles whitespace using ' (apostrophe) for whitespace to the left of the text and " (quote) for whitespace to the right of the text.

| One without
| space!
/ One withoutspace!
      
|  Space to the
|' left!
/ Space to the left!
      
|" Right
|  space here!
/ Right space here!

SYNTAX

BLOCK TEXT

A node in opulent can receive a text block, marked by the pipe symbol right after the node markup, meaning that everything written underneath, which is more indented than the node will be rendered as plain text.

p | 
  Hello Opulent!
  This is on multiple lines.
        
/ <p>
/   Hello Opulent!
/   This is on multiple lines.
/ </p>

TEXT INTERPOLATION

In opulent you can interpolate variables into your text and strings, the same way you do in Ruby.

- variable = "text" 
p This is interpolated #{variable}.
/ <p>This is interpolated text</p>

You can also interpolate variables into attribute values, whether they are wrapped, unwrapped or shorthand.

- node_class = "btn-primary"
a class="btn #{node_class}"
/ <a class="btn btn-primary"></a>
      
- node_id = "node"
a(id="#{node_id}")
/ <a id="node"></a>

- node_class = "button"
a."#{node_class}"
/ <a class="button"></a>

TEXT ESCAPE

Just like the node attributes, text is escaped by default and can be marked as unescaped using the ~ (tilde) character right at the start of the text.

p This text is <escaped>
/ <p>This text is &lt;escaped&gt;</p>
      
p ~ This text is <unescaped>
/ <p>This text is <unescaped></p>
p 
  | This text is <escaped>
/ <p>This text is &lt;escaped&gt;</p>
      
p 
  |~ This text is <unescaped>
  / <p>This text is <unescaped></p>
p |~
  This text is <escaped>
/ <p>This text is &lt;escaped&gt;</p>
      
p |~ 
  This text is <unescaped>
/ <p>This text is <unescaped></p>

The whitespace syntax has higher priority than the escape syntax.

p 
  |'"~ This text is <unescaped>
/ <p> This text is <unescaped> </p>

SYNTAX

RUBY EVAL

You can evaluate native Ruby code and pass a block into it by using the - (minus) symbol directly into your view templates.

- var = "example-"
- a = 10 
- b = 5
      
div id=(var + a.to_s)
/ <div id="example-10"></div>

SYNTAX

RUBY PRINT

You can evaluate native Ruby code and print it using the = (equals) symbol directly into your view templates.

- x = 10
- y = 5
      
= x * y 
/ 50

Evaluation print nodes can also be passed as direct inline child to any other node.

- x = "Hello "
- y = "opulent!"
      
p = x + y
/ <p>Hello opulent!</p>

SYNTAX

RUBY CODE ATTRIBUTES

Opulent can receive attribute values directly from evaluating ruby code, which you can write right after the = (equals) attribution symbol.

- div_id = "node"
      
div id=div_id
/ <div id="node"></div>

Methods

You can also call methods on a single ruby variable when using it in attributes.

- div_id = "node"
      
div id=div_id.upcase
/ <div id="NODE"></div>

Multiple Variables

If you want to use more than one variable inside an attribute, you can wrap the value inside of round brackets () (round brackets) which will allow you to apply more complex operations on the attribute's value.

- id1 = "node"
- id2 = "red"
      
div id=(id1 + "-" + id2)
/ <div id="node-red"></div>

True and False

When passing a true value to an attribute, the attribute will be rendered as a blank value attribute, which is very common in HTML5. When passing a false value to an attribute, the attribute won't be shown.

div attr=true
/ <div attr></div>
div attr=false
/ <div></div>

Arrays

You can also pass arrays as attribute values. The evaluation result will be joined with a ' ' (whitespace) character when you're using them for the class attribute and with a '_' (underline) character otherwise.

- array = ['value1', 'value2', 'value3']
      
div class=array 
/ <div class="value1 value2 value3"></div>
      
div class=['value1', 'value2', 'value3']
/ <div class="value1 value2 value3"></div>

When we're passing an array to something other than a class attribute, we'll get the following output:

- array = ['value1', 'value2', 'value3']
      
div attr=array 
/ <div attr="value1_value2_value3"></div>
      
div attr=['value1', 'value2', 'value3']
/ <div attr="value1_value2_value3"></div>

Hashes

You can also pass hashes as attribute values. The assigned attribute will serve as base name, and using each hash key, new attributes will be created.

- hash = {attr1: 'value1', attr2: 'value2'}
      
div data=hash 
/ <div data-attr1="value1" data-attr2="value2"></div>
      
div data={attr1: 'value1', attr2: 'value2'}
/ <div data-attr1="value1" data-attr2="value2"></div>

SYNTAX

BASIC INCLUDE

The include directive allows you to include external files into your current template. Files are included by copying the content in place, and they are parsed afterwards. Indentation is preserved when including a file. The .op file extension doesn't need to be specified.

/ file: definitions/example.op 
      
def node 
  .node 
    yield

To include the example.op file we have written above, we can use:

/ file: index.op
      
include definitions/example
      
node 
/ <div class="node"></div>

The include directive indentation will be preserved. In the following example, everything from inside the list.op file will be included inside of the ul node.

ul 
  include list

You can also include all opulent files from within a folder using the * GLOB modifier. However, the order will be alphabetical and dependencies might not be met correctly.

/ file: index.op
      
include definitions/*
      
node 
/ <div class="node"></div>

SYNTAX

DEFINITIONS CONCEPT

Node Definitions are reusable markup elements which provide a huge increase in development speed. Definitions allow you to group complex node markup logic inside a simple tag, and can be included in the current page from a central source.

Definitions are marked up using the def keyword and accept definition arguments with optional default values.

Definitions are to opulent what mixins are to sass and less.

def node 
  div 
      
node 
/ <div></div>

SYNTAX

DEFINITIONS ARGUMENTS

As you would expect, definitions can receive arguments, which will be available as variables with the same name. Arguments are marked up using () (round brackets) and can have a preset default value.

def node(param)
  div id=param
      
node param="example"
/ <div id="example"></div>
def node(param1, param2)
  div data-attr1=param1 data-attr2=param2
      
node param1="value1" param2="value2"
/ <div data-attr1="value1" data-attr2="value2"></div>

You can set default values for definition parameters just like you would in Ruby.

def node(attr="default")
  div data-attr=attr
      
node
/ <div data-attr="default"></div>
      
node attr="hello"
/ <div data-attr="hello"></div>

SYNTAX

DEFINITIONS EXTENSION

Definition call arguments which are not specified in the parameters list will be stored inside a Hash variable called attributes.

You can extend any node inside the definition using the + (plus) symbol.

The attributes variable is a reserved keyword in opulent definitions.

def node(param)
  div+attributes id=param  
      
node param="example" data-attr1="value" data-attr2="value"
/ <div id="example" data-attr1="value" data-attr2="value"></div>
Example 1

Attribute Manipulation

You can manipulate the attributes map before extending it.

def node(param)
  - attributes['data-attr1'].upcase!
  - attributes['data-attr2'].capitalize!
        
  div+attributes id=param  
      
node param="example" data-attr1="value" data-attr2="value"
/ <div id="example" data-attr1="VALUE" data-attr2="Value"></div>

Example 3

Bootstrap Column

Here's a practical example on how you could define a Bootstrap column element and make it much easier to use.

/ Create a boostrap column
/
def col(xs, sm, md, lg, offset = {})
  - col_class = []
  - col_class << 'col-xs-' + xs.to_s if xs
  - col_class << 'col-sm-' + sm.to_s if sm
  - col_class << 'col-md-' + md.to_s if md
  - col_class << 'col-lg-' + lg.to_s if lg
        
  - col_class << 'col-xs-offset-' + offset[:xs].to_s if offset[:xs]
  - col_class << 'col-sm-offset-' + offset[:sm].to_s if offset[:sm]
  - col_class << 'col-md-offset-' + offset[:md].to_s if offset[:md]
  - col_class << 'col-lg-offset-' + offset[:lg].to_s if offset[:lg]
        
  div+attributes class=col_class
    yield

SYNTAX

DEFINITIONS SCOPE

Local Variables

Variables declared inside of a definition have local scope, meaning they are available only inside of the definition.

- var = 3
      
def node 
  / var has a local scope here
  - var = 5
      
node
= var
/ 3, the value of var doesn't get changed

Instance Variables

Instance variables which are either declared or passed using ruby blocks in the view rendering code are available both inside and outside of definitions, meaning they have a global scope.

- @var = 3
      
def node 
  / @var instance is passed inside the definition
  - @var = 5
      
node
= @var
/ 5, the value of @var changed
